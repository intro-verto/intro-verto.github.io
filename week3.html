Третья неделя

JSON (JavaScript Object Notation) — это текстовый формат для хранения и передачи данных, независимый от языка программирования.
В JSON-объекте:
  • Ключ и значение разделяются двоеточием `:`
  • Пары ключ-значение разделяются запятыми `,`
В JSON есть два вида типов данных:
  • Примитивные: строка, число, логический (true/false), null
  • Структурные: объект, массив
  • В кавычки: строки
  • Без кавычек: числа, логические значения (true, false), null, объекты, массивы
HTTP передаёт данные в открытом виде. HTTPS шифрует данные, обеспечивая безопасность передачи.
HTTP-запрос - это сообщение от клиента серверу для выполнения действия: получить данные, отправить информацию или управлять ресурсами.

Состоит из:
  • Стартовая строка: метод, URL, версия протокола
  • Заголовки: метаданные запроса
  • Пустая строка: отделяет заголовки от тела
  • Тело (опционально): передача данных (например, JSON)
  • GET: запрос данных
  • POST: отправка данных на сервер
  • PUT: создание/замена ресурса
  • PATCH: частичное обновление ресурса
  • DELETE: удаление ресурса
  • PUT: заменяет весь ресурс. Пример: `{ "name": "John", "age": 30 }`
  • PATCH: изменяет часть ресурса. Пример: `{ "age": 30 }`
Дополнительные параметры в URL после `?`. Формат: `key=value`, разделяются через `&`. Пример: `/search?query=test&page=1`.
Метод является идемпотентным, если повторный запрос не изменяет состояние системы после первого выполнения.
  • POST — создаёт новый ресурс или вызывает действие, изменяющее состояние
  • PATCH — частично обновляет ресурс, и результат может зависеть от каждого выполнения
  • При переходе на главную страницу сайта: GET — для запроса ресурса
  • При авторизации на сайте: POST — для отправки данных (логин и пароль)
  • При редактировании даты рождения: PATCH — для частичного изменения ресурса
  • 1xx — Информационные: запрос получен, обработка продолжается
  • 2xx — Успешные: запрос выполнен успешно
  • 3xx — Перенаправления: серверу нужно перенаправить клиента
  • 4xx — Ошибки клиента: некорректный запрос к серверу
  • 5xx — Ошибки сервера: сервер не может выполнить запрос
200 (OK) — запрос выполнен успешно. Этот код возвращается при успешном переходе на страницу сайта.
201 (Created) — создан новый ресурс (например, после POST-запроса). Этот код возвращается при успешной регистрации пользователя на сайте.
301 (Moved Permanently) — ресурс перемещен навсегда на новый URL.
400 (Bad Request) — запрос неправильный (ошибка синтаксиса, неверные данные).
401 (Unauthorized) — требуется авторизация.
403 (Forbidden) — доступ к ресурсу запрещен.
404 (Not Found) — ресурс не найден. Этот код возвращается при переходе на несуществующую страницу.
500 (Internal Server Error) — ошибка сервера. Это код, который меньше всего хочется увидеть на главной странице сайта после релиза.
502 (Bad Gateway) — некорректный ответ от шлюза или прокси-сервера.
503 (Service Unavailable) — сервис временно недоступен (например, из-за перегрузки).
